⚙️ Layer 1 — Infrastructure

🗄️
snowflake/setup/
Data Warehouse Configuration
  • SQL
    01_databases.sql
    Creates the ULEZ_DB database and BRONZE, SILVER, GOLD schemas
  • SQL
    02_warehouses.sql
    Creates the compute warehouse for query processing
  • SQL
    03_roles.sql
    Permissions and access control (RBAC)
  • SQL
    05_streaming_setup.sql
    Configures Streams and Tasks for pipeline automation
📦
dbt_project/
dbt Configuration
  • YML
    dbt_project.yml
    Defines the project name, version, and connection profile
  • YML
    models/staging/sources.yml
    Declares source tables in the BRONZE schema (audi_raw, bmw_raw, ford_raw...)
Snowflake hosts tables → Python scripts ingest data

🥉 Layer 2 — Bronze (Raw Data Ingestion)

🔌
scripts/
Ingestion Engine (Python)
  • PY
    autotrader_collector.py
    Connects to AutoTrader's GraphQL API and extracts car listings
  • PY
    data_engine.py
    Orchestrates the pipeline: collect → upload to Snowflake BRONZE
  • PY
    check_counts.py
    Validates how many rows were loaded into the warehouse

These scripts write to the BRONZE.autotrader_raw table.
Data arrives dirty: commas in prices, null fields, wrong types.

📂
data/raw/
Local Raw Data (Backup / CSV)
  • CSV
    *.csv
    Backup CSV files with listings per brand (audi, bmw, ford...)

Serves as a fallback in case the API is offline.
Also used for local testing and validation.

dbt run → Transforms Bronze into Silver (Cleansing & Type Casting)

🥈 Layer 3 — Silver (Cleansing & Standardisation)

🧹
dbt_project/models/staging/
Staging Layer (dbt Views)
  • SQL
    stg_cars.sql
    Reads from BRONZE.autotrader_raw and standardises all fields

What happens here?

brand → Extracted from title using UPPER()
price → Converted from text "£12,500" to number 12500.00
fuel_type → Renamed from "fuelType" to snake_case standard
engine_size → Renamed from "engineSize"

⚡ Materialised as a VIEW — does not store data, transforms on-the-fly at query time.

dbt run → Transforms Silver into Gold (Business Logic + Aggregation)

🥇 Layer 4 — Gold (Facts, Dimensions & Analytics)

⭐
dbt_project/models/marts/fct_cars.sql
Enriched Fact Table (Wide Table)
  • FK
    Reads from: {{ ref('stg_cars') }}
    References the Silver layer above

DIMENSION Columns (Context)

brand DIM — Car manufacturer
model DIM — Car model
fuel_type DIM — Petrol / Diesel / Electric
year DIM — Year of manufacture
transmission DIM — Manual / Automatic
is_ulez_compliant DIM — ULEZ Compliant? (Business Rule)

FACT Columns (Metrics)

price FACT — Car price
mileage FACT — Odometer reading
vehicle_age FACT — Calculated age (2025 - year)
price_per_mile FACT — Value per mile driven

📊
dbt_project/models/marts/mart_ulez_impact.sql
Data Mart for Dashboard Consumption
  • FK
    Reads from: {{ ref('fct_cars') }}
    References the Fact Table on the left

Grouped by DIMENSIONS:

GROUP BY brand, year, fuel_type, is_ulez_compliant

AGGREGATED Metrics:

total_listings AGG — COUNT(*)
avg_price AGG — AVG(price)
avg_mileage AGG — AVG(mileage)
avg_age AGG — AVG(vehicle_age)
price_efficiency AGG — price / age

Python reads Gold data → Trains ML models
Gold + ML feed the Dashboard

🧠 Layer 5 — Machine Learning & Dashboard

🤖
ml_analysis/
Machine Learning Models
  • PY
    price_model.py
    Regression model to predict price impact based on ULEZ compliance status

Consumes FACTS (price, mileage) grouped by DIMENSIONS (fuel_type, is_ulez_compliant) to train the model.

🔬
scripts/ml_clustering.py
K-Means Clustering
  • PY
    ml_clustering.py
    Segments the market into clusters using Scikit-Learn and writes results back to Snowflake

Writes a new cluster_id column back to the Gold layer.

📱
app/
Streamlit Dashboard
  • PY
    app.py
    Interactive dashboard that reads directly from GOLD.mart_ulez_impact and GOLD.fct_cars to render charts

The dashboard uses DIMENSIONS as filters (brand dropdown, fuel type selector) and FACTS as chart values (average price bars, mileage distributions).
